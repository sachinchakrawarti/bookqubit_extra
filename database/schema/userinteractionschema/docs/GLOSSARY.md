# User Interaction Schema Glossary

## Overview

This glossary defines the common terms, concepts, abbreviations, and database terminology used throughout the **User Interaction Schema** documentation for the BookQubit platform.

The purpose of this document is to provide a consistent vocabulary for developers, database administrators, API designers, and contributors.

---

# A

## API

**Application Programming Interface**

A set of endpoints that allows applications to communicate with the BookQubit backend.

Example:

```text
GET /api/user-reviews
```

---

## Archive

The process of moving older or inactive records to long-term storage while preserving them for future reference.

---

## Audit Log

A record of important actions performed by users or the system.

Examples:

- Login
- Review creation
- Rating update
- Permission change

---

## Authentication

The process of verifying the identity of a user or system before granting access.

Examples:

- JWT
- OAuth
- Session Login

---

## Authorization

The process of determining what an authenticated user is allowed to access or modify.

---

# B

## Backup

A copy of the database or selected data that can be restored after data loss or system failure.

---

## Bookmark

A saved reference allowing users to quickly return to a book.

---

## Bookshelf

A user-managed collection used to organize books into custom groups.

Examples:

- Favorites
- Want to Read
- Completed
- Science Books

---

## Batch Operation

Executing multiple database operations together to improve efficiency.

---

# C

## Cache

Temporary storage used to speed up repeated data access.

Examples:

- Memory cache
- Redis
- Browser cache

---

## Collection

A user-created grouping of books organized around a topic, goal, or personal preference.

---

## Constraint

A database rule that enforces data integrity.

Examples:

- PRIMARY KEY
- FOREIGN KEY
- UNIQUE
- CHECK
- NOT NULL

---

## CRUD

The four basic database operations:

- Create
- Read
- Update
- Delete

---

## Cursor Pagination

A pagination technique that uses a reference value (cursor) instead of offsets, improving performance for large datasets.

---

# D

## Database Schema

The logical structure of a database, including tables, columns, relationships, indexes, views, and constraints.

---

## Documentation

Technical information describing the schema, APIs, migrations, and development practices.

---

# E

## ER Diagram

**Entity Relationship Diagram**

A visual representation of tables and their relationships.

---

## Endpoint

A URL that provides access to a specific API resource.

Example:

```text
/api/user-bookmarks
```

---

# F

## Foreign Key

A column that references the primary key of another table, maintaining referential integrity.

---

## Full-Text Search (FTS)

A specialized search capability for efficiently querying large text fields.

Examples:

- SQLite FTS5
- PostgreSQL Full Text Search
- MySQL FULLTEXT

---

# G

## Git

A distributed version control system used to manage source code and documentation.

---

## GraphQL

A query language and API technology that allows clients to request only the data they need.

---

# H

## Highlight

A user-selected section of a book saved for future reference.

---

## HTTP

**Hypertext Transfer Protocol**

The protocol used for communication between clients and servers.

---

# I

## Index

A database structure that improves query performance by allowing faster data lookup.

Example:

```text
idx_user_reviews_book
```

---

## Integrity

The accuracy, consistency, and reliability of data stored in the database.

---

# J

## JSON

**JavaScript Object Notation**

A lightweight data format commonly used for API responses and seed files.

Example:

```json
{
  "user_id": 10,
  "book_id": 25
}
```

---

## JWT

**JSON Web Token**

A compact token format commonly used for stateless authentication.

---

# L

## Log

A record of events generated by the application or database.

---

# M

## Migration

A version-controlled change to the database schema.

Examples:

- Create table
- Add column
- Create index
- Update constraint

---

## Metadata

Information that describes other data.

Examples:

- created_at
- updated_at
- version
- status

---

# N

## Normalization

A database design process that reduces redundancy and improves data consistency.

---

## Notification

A message delivered to a user regarding platform events or updates.

---

# O

## Offset Pagination

A pagination method using `LIMIT` and `OFFSET` to retrieve subsets of records.

---

# P

## Pagination

The process of dividing large datasets into smaller pages.

---

## Primary Key

A unique identifier for each record in a table.

Example:

```text
review_id
```

---

## Prepared Statement

A parameterized SQL query that helps improve security and performance by separating SQL logic from user input.

---

# Q

## Query

A request to retrieve or modify data in the database.

Example:

```sql
SELECT * FROM user_reviews;
```

---

# R

## Rating

A numeric score assigned by a user to a book.

---

## RBAC

**Role-Based Access Control**

A permission model based on predefined user roles.

---

## README

The primary documentation file describing a project or module.

---

## Relationship

A logical association between database tables.

Examples:

- One-to-One
- One-to-Many
- Many-to-Many

---

## REST API

An API style that uses HTTP methods and resource-based URLs.

---

## Review

A user's written opinion or evaluation of a book.

---

# S

## Schema

The organizational structure of a database.

---

## Search History

A record of previous search queries made by a user.

---

## Seed Data

Sample or initial data used to populate database tables.

---

## Soft Delete

A technique where records are marked as deleted (for example, with `deleted_at`) instead of being permanently removed.

---

## SQL

**Structured Query Language**

The standard language for managing relational databases.

---

## SQLite

A lightweight, serverless relational database engine.

---

# T

## Table

A collection of related rows and columns representing a specific type of data.

---

## Timestamp

A field storing date and time information.

Examples:

- created_at
- updated_at
- deleted_at

---

## Transaction

A sequence of database operations executed as a single unit of work that either fully succeeds or fully fails.

---

## Trigger

A database object that automatically executes when specified events occur.

---

# U

## Unique Constraint

A rule ensuring that duplicate values cannot exist in specified columns.

---

## User Interaction

Any action performed by a user within the platform.

Examples:

- Rating
- Bookmarking
- Reviewing
- Highlighting
- Searching
- Reading

---

# V

## Validation

The process of verifying that data meets defined rules before being stored or processed.

---

## Versioning

Tracking and managing changes to the database schema, APIs, and documentation over time.

---

## View

A virtual table generated from the result of a SQL query.

---

# W

## WAL

**Write-Ahead Logging**

A SQLite journal mode that improves concurrency and reliability.

---

## Webhook

An HTTP callback used to notify external systems about events.

---
